Requirements

Ensure the following components are installed before loading Structures Overhaul in your world or modpack.

Mod loader

Forge or NeoForge 1.20.1 with latest recommended build. Older versions are not supported.

  • Java 17 runtime
  • Create 0.5.1+
  • Flywheel 0.6+

Client & server

Install on both sides for dedicated servers. Copy the exact same jar to avoid desync and structure mismatches.

Server operators should pre-generate worlds to validate biome compatibility.

Optional libraries

Kotlin for Forge provides scripting hooks used by some addons. Install it if you plan to load integration packs.

Installation steps

  1. Download the latest release.

    Grab the Create Structures Overhaul jar from Modrinth, CurseForge, or the GitHub releases page.

  2. Place the jar in /mods.

    Keep the file name intact. If you are updating from an older version, remove the previous jar to avoid duplicate structures.

  3. Copy the default datapacks.

    The archive includes optional datapacks that expose worldgen presets and loot tables. Drop them into the /datapacks folder for each save.

  4. Update configuration files.

    Launch once to generate config/create-structures-overhaul, then edit spawn weights, biome tags, or loot pools to match your pack.

Verify the setup

After launching the game, confirm the mod loaded correctly.

  • Open the in-game mod list and ensure Create: Structures Overhaul appears without warnings.
  • Run the /cso locate ashen_foundry command in a creative test world to confirm the structure registry is active.
  • If you are using a server, check the console for the Generated structure palette registry log entry.
Tip: add the provided cso_dev datapack when testing new structure packs to see debug bounding boxes.

Troubleshooting

If the game fails to load or no structures appear, walk through the following checklist.

Common issues

  • Duplicate mods: ensure only one copy of the jar exists in /mods.
  • Missing dependencies: update Create, Flywheel, and Kotlin to their latest versions.
  • Unsupported world version: the add-on targets Minecraft 1.20.1 only.
  • Biome conflicts: adjust datapack tags if other mods replace vanilla biome IDs.
Still stuck? Collect your latest.log and open an issue on GitHub with the installation label.
